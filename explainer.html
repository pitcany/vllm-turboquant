1 The Problem

KV cache grows with every token

During generation, every layer stores a Key and Value vector for every previous token. For long contexts this dominates GPU memory.

Token 1 → K₁ V₁ Token 2 → K₁ V₁ K₂ V₂ Token 3 → K₁ V₁ K₂ V₂ K₃ V₃ ... Token N → K₁..Kₙ V₁..Vₙ ← O(N) memory

Qwen3.5-27B on 4× RTX 3090

  • Model weights: ~13 GB / GPU
  • KV cache: ~7.5 GB / GPU (30 GB total)
  • Max context: 457,072 tokens
  • Want longer? Need more GPUs — expensive

What if we could compress the KV cache to 39% of its size?

3 Key Compression (3-bit)

Step A: Random Rotation

Multiply keys by a random orthogonal matrix Π. This spreads information uniformly across dimensions so no single dimension carries disproportionate signal.

Krot = K · Π    (norm-preserving)

Step B: Lloyd-Max Quantization

After rotation, each dimension follows a Beta distribution. Lloyd-Max finds the MSE-optimal 8 centroids for this distribution.

Codebooks are pre-computed offline for d=256 — zero runtime overhead.

Step C: QJL Residual Signs

The quantization residual r = Krot - K̂ still contains directional information. QJL (Quantized Johnson-Lindenstrauss) projects it into a compact sign vector:

signs = sign(S · r)

where S is a random ±1 matrix. This preserves inner-product relationships with high probability.

4 Value Compression (2-bit)

Group Quantization

Values are split into groups of 32 dimensions. Each group gets its own scale and zero point, then each value is mapped to one of 4 levels (2 bits).

For group [d₀ .. d₃₁]: scale = (max - min) / 3 zero = min idx_i = round((val_i - zero) / scale) ← 0,1,2,3 Bit-packing (4 values per byte): byte = idx₀ | (idx₁<<2) | (idx₂<<4) | (idx₃<<6) 256 dims → 64 packed bytes + 16B scales + 16B zeros = 88 bytes

Why 2-bit works for values

  • Values are summed with softmax weights during attention — small errors average out across many tokens
  • Per-group scales preserve the dynamic range of each dimension cluster
  • 4× denser than 8-bit quantization

6 Fused Triton Decode Kernel

What happens every decode step

For each new query token q, against all N cached tokens:

  1. Dequantize keys: indices → centroids, undo rotation via ΠT
  2. Attention scores: αi = q · ki / √d
  3. Dequantize values: unpack 2-bit → scale · idx + zero
  4. Weighted sum: out = ∑ softmax(αi) · vi

All 4 steps fused into ONE Triton kernel — no intermediate tensors allocated.

Why zero overhead

  • Decode is compute-bound, not memory-bound
  • Reading compressed data is faster (less memory bandwidth)
  • Dequantization is cheap ALU ops fused into the attention loop
  • Works with GQA (Grouped-Query Attention)
