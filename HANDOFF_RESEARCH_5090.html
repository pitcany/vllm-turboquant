10) Known blockers / failure modes

  1. Interrupted runs leave GPU processes alive
  2. cancellation can leave a Python process attached to GPU using ~28 GB VRAM.
  3. Baseline 200k not practical in current config
  4. repeated timeout/stall behavior.
  5. Host/process kill (exit 137) during long scripted telemetry
  6. long single-process “collect everything” jobs can be killed before final JSON write.
  7. FP8 KV + TurboQuant stacking
  8. still incompatible in earlier testing path (FlashInfer metadata mismatch).

11) Operational playbook (researcher)

A) Always clean GPU before runs

python3 - <<'PY'
import os, subprocess
try:
    out=subprocess.check_output(['nvidia-smi','--query-compute-apps=pid','--format=csv,noheader,nounits'], text=True)
    for x in out.splitlines():
        x=x.strip()
        if x:
            try: os.kill(int(x),9)
            except: pass
except: pass
print(subprocess.check_output(['nvidia-smi','--query-gpu=memory.used,memory.free,utilization.gpu','--format=csv,noheader,nounits'], text=True))
PY
